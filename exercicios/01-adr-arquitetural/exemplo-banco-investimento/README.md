# Exemplo — ADR-0001 Stack mobile para banco de investimento

> **Não é entrega de aluno.** É exemplo do prof pra mostrar formato esperado.
> Diretório fora do padrão `aluno-*` pra não acionar o autograder.

Caso fictício: banco digital com app de investimento retail B2C, escala 2M→5M usuários, compliance BACEN, time 12 engs, janela 9 meses.

## Por que é um exemplo bom

- **Contexto enxuto** (1 parágrafo + bullets) — não dá rolezinho
- **Matriz quantitativa** com critérios + pesos + notas + score calculado
- **5 alternativas** analisadas, não só as 2 favoritas
- **Consequências honestas** — diz o que vai doer
- **Mitigações** explícitas pros pontos negativos (raciocínio, não otimismo)
- **5 referências** com mix correto: 1 paper CACM + 1 livro O'Reilly + 1 post engenharia + 1 norma regulatória + 1 fonte primária (Nygard)

## O que evitar

- Copiar este exemplo trocando nomes — fica fácil de pegar
- "Performance: 8" sem dizer por quê (justifique cada nota no parágrafo Decisão)
- Alternativa só pra inflar lista (PWA-wrapper aqui foi avaliado de verdade, não palha)
- "Vai dar tudo certo" — ADR sem consequências negativas vira marketing
